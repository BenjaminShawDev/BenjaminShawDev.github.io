Final Year AI Projects

For each of these projects the base applications were provided by the University without AI with the goal to implement AI that performed adequately. I did not create any of these programs, I only added the AI to them.

Neural Network - Flappy Bird

Originally this was a remade version of Flappy Bird that had to be played manually, Flappy Bird being a game where clicking causes the bird to "flap" making it move upwards with the goal to dodge incoming pipes. For this task a multi-layer perceptron neural network was created that took three inputs, had one hidden layer consisting of three nodes and had one output. The three inputs were distance from the floor, distance to the next pipe and the height of the gap. The output node was whether to flap or not.

A genetic algorithm was used to train the network with the run's time being used as the fitness value. To speed up the training of the network the code was edited to allow for an entire generation of birds to do a run simultaneously.

The end result was a bird that could survive indefinitely.

Genetic Algorithm - Tower Defence

The base game provided continually spawned waves of "enemies" with the player being able to place three types of towers around the path that the enemies followed. The towers consisted of an expensive one that did a lot of damage but attacked slowly and had a small range, a slightly cheaper one that did a small amount of damage but had a fast attack rate and large attack radius. The last tower was inbetween the other two.

The goal of the genetic algorithm was for it to eventually reach a score of 200 which it managed to do, it was also done with multiple types of selection and crossover types. The genes consisted of three numbers representing an X and Y position and a tower type, the game would then try to place each of these towers accordingly. These genes were mutated between runs by randomising 10% of them in each chromosome. The elitist selection type ended up resulting in the best scoring run.

MiniMax Algorithm

Starting off with a simple 3D program that knew the rules of chess the aim was to utilise the Minimax algorithm for it to be able to play chess competently and with fast responses. The Minimax algorithm theoretically looks at every possible outcome, scores each possible state of the game and chooses the best outcome. Realistically looking at every possible outcome would take an extremely long time so for this program it was capped at a depth of three. It creates a tree structure with each depth alternating which player's turn it is. On the AI's turn it picks the maximum score, on the opponent's turn it picks the minimum score to represent the opponent doing the best move against the AI.

Alpha-beta pruning was also added to make it avoid going down nodes on the tree that could not produce a better outcome than the one that it already selected. This resulted in increased efficiency without changing the outcome. Some opening moves were also implemented to increase efficiency initially and a bit more intelligent. Once all this had been done it could beat people at chess without having to wait for it to think between turns.

Artificial Life - Flocking

This program focused on boids, a type of artifical life, where independent agents simulate flcoking behaviours as seen in birds and fish. By calculating alignment, cohesion and separation values for each boid based on nearby boids it can result in all the boids moving together without the need for communication between them. A predator was introduced to see if flcoking improved the flock's survival.

Each boid also had slightly randomised values for attributes like movement speed and sight range. The effect of the alignment, cohesion and separation values could also be altered.
