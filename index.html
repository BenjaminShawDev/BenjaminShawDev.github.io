    Pacman Game

    A simple game with the theme of Pacman but plays slightly differently

    The 50 munchies are spread out randomly and are put through a for loop to detect if any overlap and if any do they are moved to another location on the screen.

    There is a box that follows the mouse, the box can detect collisions via AABB collision, if the box touches a ghost the ghost is sent in the opposite direction, and while the box collides with Pacman it disappears.

    If you pick up the skull the ghosts change behaviour, instead of travelling up or across they instead move towards Pacman by comparing their X and Y coordinates with Pacman's and move accordingly.

    Mario Game

    A short and simple Mario game made in SDL consisting of two levels and two players which are both controlled using a single keyboard.

    The collision for the terrain in each level is determined by a simple bitmap and the player sprite positions are checked against it to see if they are touching something which should be solid or not and then acts accordingly.

    This project is the first time I had to think about changing levels so I chose to recreate going down a pipe by detecting whether a player was above it and then moved them down the screen and used a Y-coordinate check on the sprites to switch the level.

    Asteroid Game

    A 3D game made in OpenGL where you get to control a space ship in an asteroid field.

    This was my first 3D project and it really helped me understand the basics of OpenGL and creating a game in a 3D space.

    There are a set amount of asteroids in the game and when they go behind the camera they are translated back to the front of the game and with the use a random number generator are given random X and Y coordinates.

    Collision detection was done by taking the position of the player and each asteroid and using 3D Pythagoras' thereom to calculate the distance between them and if it was smaller than the radius of the ship it would class as a collision.

    Knight Game

    A simple SDL game I made in the summer of 2020 which has no objectives but was used to experiment with SDL.

    The levels are generated by taking the pixel colour of each pixel from the map's PNG file and converting it into a bitmap. If the pixel colour matched the requirements for a tile that tile would be placed in the corresponding place in the game.

    By also learning from an online tutorial it helped me create an Entity Component System which helped with moving the sprites around with the camera and adding properties like collision to the water tiles.

    The camera edge collision was done by creating a rectangle the size of the viewport, the rectangle would detect when it has gone outside the specifed boundaries and stop the camera at the edge of the screen.

    The pause and options menu was the first time I had tried creating my own user interface and I added basic mouse support by creating a mouse collision function that detected when the mouse was over a highlighted option.

    Network Application

    A chat application that sends packets to and from a server to multiple clients and utilises multithreading.

    Using Windows Forms I created a basic GUI which easily allows the user to send packets containing messages to all the other clients or a specific client.

    The server and each of the clients run on their own threads to reduce concurrency issues like freezing until a server response is received and it helps improve performance.

    There also is a paint window which allows the clients to paint on the same canvas by sending packets with which pixels coordinates had been drawn on and with what colour.

    DirectX Framework

    A framework created with DirectX 11 in which the user can create and transform different textured 3D objects.

    The user can move around a 3D space and look around with basic mouse input with the option to switch between different cameras, which are each stored in an array, for different perspectives.

    Pressing the left mouse button creates textured objects in front of them which are loaded from OBJ and DDS files. The object that is loaded can be changed and so can its material.

    Pixel shading is implemented with the use of a shader file which is affected by the object material and the colour of the light can be changed as well.
