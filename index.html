    Pacman Game

    A simple game with the theme of Pacman but plays slightly differently

    The 50 munchies are spread out randomly and are put through a for loop to detect if any overlap and if any do they are moved to another location on the screen

    There is a box that follows the mouse, the box can detect collisions via AABB collision, if the box touches a ghost the ghost is sent in the opposite direction, and while the box collides with Pacman it disappears

    If you pick up the skull the ghosts change behaviour, instead of travelling up or across they instead move towards Pacman by comparing their X and Y coordinates with Pacman's and move accordingly

    Mario Game

    A short and simple Mario game made in SDL consisting of two levels and two players controlled via a single keyboard

    The collision for the level terrain are determined by a simple bitmap for each level and the player sprites positions are checked against it

    This project is the first time I had to think about changing levels so I chose to recreate going down a pipe by detecting whether a player was above it and then moved them down the screen and used a Y-coordinate check to switch the level

    Github link

    Asteroid Game

    A 3D game made in OpenGL where you get to control a space ship in an asteroid field

    My first 3D project and it really helped me understand the basics of OpenGL and creating a game in a 3D space

    There are a set amount of asteroids in the game and as soon as they go behind the camera they are sent back to the front of the game with random X and Y coordinates

    I did collision detection by taking the position of the player and every asteroid each frame and using 3D Pythagoras' thereom to work out the distance between them and if it was smaller than a certain amount it would class as a collision

    Github link

    Knight Game

    A simple SDL game I made over the 2020 summer which has no objectives but was used to experiment with SDL

    The levels are generated from a PNG by taking the pixel colour of each pixel from the map file and converting it into a bitmap, if the pixel colour matched a tile that tile would be placed in the corresponding place in game

    By also learning from an online tutorial it helped me create an entity component system which helped with moving the sprites around with the camera and adding properties like collision to the water tiles

    The camera edge collision was done by creating a rectangle the size of the view port, the rectangle would detect when it has gone outside the specifed boundaries and stop the camera at the edge of the screen

    The pause and options menu was the first time I had tried creating my own user interface and I added basic mouse support by creating a mouse collision function that detected when the mouse was over a highlighted option
